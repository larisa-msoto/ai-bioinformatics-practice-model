# Discernment - Practice Model

# Evaluating the biological plausibility of AI outputs.

## Competencies by Area

* Biological integrity verification
    * Verify the biological plausibility of AI-generated results against established mechanistic principles in your specific life science domain, and trusted reference sources or databases.

* Statistical auditing
    * Assess the statistical assumptions, p-values, and error controls of AI-generated results to verify mathematical rigor.

* Biological interpretation and contextualization
    * Synthesize AI-generated results with established knowledge to extract meaningful biological insights and draw valid conclusions.

* ACTIVE: Critical Verification Protocols
    * Implementing structured validation procedures to audit AI-generated code or biological hypotheses.

---

# Benchmarking AI results against standard baselines

## Competencies by Area

* Algorithmic bias detection
    * Screen AI outputs for population skews, data imbalances, or training data biases across diverse experimental groups.

* Automated evaluation design
    * Build programmatic evaluation pipelines that automatically calculate error rates and compare AI outputs against gold-standard biological datasets or software tools.

* ACTIVE: Verification Confidence Calibration
    * Closing the "proof-belief gap" by aligning one's belief in an AI solution's correctness with objective accuracy outcomes.

---

# Detecting computational artifacts and hallucinations

## Competencies by Area

* Hallucination detection
    * Identify fabricated data, non-existent literature citations, or flawed logical loops within AI text and code outputs.

* Collaborative code review
    * Engage in structured peer code reviews, pull requests, and issue tracking on GitHub to systematically verify the accuracy and logic of AI-generated code in an iterative manner throughout the project's lifecycle.

* Code auditing and debugging
    * Audit AI-generated scripts line-by-line to expose syntax errors, silent logical bugs, or hardcoded variables before running them.
